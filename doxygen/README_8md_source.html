1 # playd
2 
3 `playd` (_player daemon_) is a C++ audio
4 player developed by [University Radio York]. It's designed to be
5 minimal, hackable, and composable into bigger systems (such as our
6 ongoing BAPS3 project to build a new radio playout system).
7 
8 
9 ## Usage
10 
11 `playd DEVICE-ID [ADDRESS] [PORT]`
12 
13 * Invoking `playd` with no arguments lists the various device IDs
14  available to it.
15 * Full protocol information is available on the GitHub wiki.
16 * On POSIX systems, see the enclosed man page.
17 
18 For command usage, see `README.commands.md`.
19 
20 ### Sending commands manually
21 
22 To connect directly to `playd` and issue commands to it, you can use
23 [netcat]:
24 
25  # If you specified [ADDRESS] or [PORT], replace localhost and 1350 respectively.
26  $ nc localhost 1350
27 
28 On Windows, using [PuTTY] in _raw mode_ (__not__ Telnet mode) with
29 _Implicit CR in every LF_ switched on in the _Terminal_ options should
30 work.
31 
32 __Do _not_ use a Telnet client (or PuTTY in telnet mode)!__ `playd` will
33 do weird things in the presence of Telnet-isms.
34 
35 
36 ## Features
37 
38 * Plays MP3s, Ogg Vorbis, FLACs and WAV files;
39 * Seek;
40 * Frequently announces the current position;
41 * TCP/IP interface with text protocol;
42 * Deliberately not much else.
43 
44 
45 ## Philosophy
46 
47 `playd` is developed using the following guidelines:
48 
49 * Do one thing and do it well;
50 * Be hackable;
51 * Favour simplicity over performance;
52 * Favour simplicity over features;
53 * Let other programs handle the shinies.

56 ## Compilation
57 
58 ### Requirements
59 
60 * [libuv] 1.9.1+
61 * [SDL2] 2.0.3
62 * A C++14 compiler (recent versions of [clang], [gcc], and Visual Studio
63  work)
64 * [CMake] 2.8+, for building.
65 
66 The following dependencies are used for file format support, and you'll need at
67 least one of them:
68 
69 * [libmpg123] 1.20.1+, for MP3 support;
70 * [libflac++] 1.3.0+, for FLAC support;
71 * [libsndfile] 1.0.25+, for Ogg Vorbis, WAV, and FLAC support (if libflac++ isn't available).
72 
73 Certain operating systems may need additional dependencies; see the OS-specific
74 build instructions below.
75 
76 ### CMake
77 
78 `playd` can be built with cmake: just run `cmake .` in the root directory
79 (with whichever additional flags you need: run `cmake --help` for additional
80 information), then use your favourite build tool (`make`, Visual Studio, etc.)
81 with the generated projects.
82 
83 ### POSIX (GNU/Linux, BSD, OS X)
84 
85 **Warning**: This method of building `playd` is liable to be removed in
86 favour of cmake in future.
87 
88 `playd` comes with `config.sh`, a Bourne shell script that will generate a
89 GNU-compatible Makefile that can be used both to make and install.
90 
91 To use the Makefile, you'll need [GNU Make] and `pkg-config` (or equivalent),
92 and pkg-config packages for SDL2, libuv, and any needed decoder libraries.
93 We've tested building playd on Gentoo, FreeBSD 10, and OS X, but other
94 POSIX-style operating systems should work.
95 
96 Using the Makefile is straightforward:
97 
98 * Ensure you have the dependencies above;
99 * Run `config.sh` (optionally, read it first to see if any variables need to be
100  overriden for your environment);
101 * Optionally read the generated `Makefile`, to make sure it's ok;
102 * Run `make` (or whatever GNU Make is called on your OS; in FreeBSD, for
103  example, it'd be `gmake`), and, optionally, `sudo make install`.
104  The latter will globally install playd and its man page.
105 
106 #### OS X
107 
108 All dependencies are available in [homebrew] - it is highly recommended that
109 you use it!
110 
111 #### FreeBSD (10+)
112 
113 FreeBSD 10 and above come with `clang` 3.3 as standard, which should be able to
114 compile `playd`. `gcc` is available through the FreeBSD Ports Collection
115 and package repositories.
116 
117 You will need `gmake`, as `Makefile` is incompatible with BSD make. Sorry!
118 
119 All of `playd`'s dependencies are available through both the FreeBSD Ports
120 Collection and standard package repository. To install them as packages:
121 
122  root@freebsd:/ # pkg install gmake libmpg123 libsndfile libflac libuv sdl2 pkgconf
123 
124 Then, run `gmake` (__not__ `make`), and, optionally, `gmake install` to install
125 `playd` (as root):
126 
127  user@freebsd:~/ % gmake
128  root@freebsd:~/ # gmake install
129 
130 ### Windows
131 
132 #### Visual Studio
133 
134 **Warning**: This method of building `playd` is liable to be removed in
135 favour of cmake in future.
136 
137 _For more information, see `README.VisualStudio.md`._
138 
139 playd **can** be built with Visual Studio (tested with 2015 Community), but
140 you will need to source and configure the dependencies manually. A Visual
141 Studio project is provided, but will need tweaking for your environment.
142 
143 #### MinGW
144 
145 We haven't managed ourselves, but assuming you can build all the dependencies,
146 (libsox is the difficult one), it should work fine.
